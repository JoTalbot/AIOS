# AIOS 9.0.0-alpha.7 — Scaffold, Fleet, PoolMonitor

**Дата:** 2026-07-21 · **Тесты:** 705/705 зелёные · Python ≥ 3.10

Восьмой выпуск добавляет кодогенерацию платформ, автоматическое
обеспечение профилей устройствами и демон живости пула.

## Главное

### `aios platforms scaffold` — платформа одной командой
```bash
aios platforms scaffold --name prom-ua --package ua.prom.app
```
Генерирует: `platforms/<name>.yaml` (дескриптор каталога), модуль
`aios_core/modules/<name>/` с хранилищем — наследником OLXStorage
(вся схема: объявления, история цен, outbox, свои объявления,
конкурентные связи — доступна сразу) и smoke-тест. После
`load_catalog()` платформа сразу работает во всех общих surfaces:
профили, пул устройств, MCP. `--dry-run` показывает план. Валидация
имени/пакета, защита от перезаписи существующих файлов.

### `ensure_device` — связка bootstrap × pool
Профилю гарантируется устройство:
1. идемпотентная аренда свободного устройства из пула;
2. при нехватке — `avdmanager create avd`, запуск headless-эмулятора,
   ожидание загрузки, регистрация serial в пуле и аренда профилю
   (с синхронизацией в реестр профилей).

Все побочные эффекты — инъецируемые функции, тесты не требуют Android
SDK. CLI: `aios devices ensure --profile olx:work`.

### PoolMonitor — демон живости пула
Опрашивает `adb devices`, шлёт heartbeats за живые серийники,
периодически переводит молчащих в `offline` с освобождением аренд.
CLI: `aios devices monitor [--interval N]` (демон) или `--once`
(один цикл — формат для cron/systemd-timer).
