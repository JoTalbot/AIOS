# AIOS — Глоссарий

Единый словарь терминов AIOS. Используется во всех редакциях документации
(`docs/aios/`) независимо от версии.

| Термин | Определение |
|---|---|
| **AIOS** | Self-evolving distributed OS для прикладного интеллекта, автотестирования, генерации API, эволюции навыков и коллективного обучения. |
| **Конституция** | Набор неизменяемых принципов, ограничивающих действия системы (см. `docs/aios/constitution/`). |
| **ConstitutionEngine** | Движок оценки действия по конституции; возвращает ALLOW / REVIEW / DENY. |
| **DecisionEngine** | Принимает решение, комбинируя конституционный вердикт и обученный опыт. |
| **ApprovalManager** | Управляет человеческим подтверждением критических действий. |
| **Memory** | Хранилище опыта и знаний по категориям: personal / operational / constitutional. |
| **Knowledge Graph** | Граф связей между правилами, знаниями и опытом. |
| **LearningEngine** | Извлекает повторяемые паттерны из успешного опыта. |
| **ConsensusEngine** | Распределённое согласование предложений взвешенным голосованием. |
| **EvolutionManager** | Управляет безопасным жизненным циклом изменений системы. |
| **Orchestrator** | Интеграционная точка: связывает движки в цикл perceive→decide→remember→consensus→execute→evolve. |
| **Federation** | Объединение узлов AIOS в распределённую сеть (глобальная федерация). |
| **Autonomy** | Способность системы действовать без внешнего управления в рамках конституции. |
| **Governance** | Прозрачность, аудит и человеческий надзор над эволюцией системы. |
| **Policy** | Декларативное правило в `policies/*.yaml`, загружаемое `PolicyLoader`. |
